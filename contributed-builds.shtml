Contributed builds of Pale Moon

The following list is a list of endorsed third party builds of Pale Moon. If the browser is not in this list, it is not an authorized redistribution/build. Please note that these third-party builds may lag behind the main-line releases of Pale Moon.

Alternate operating systems/configurations:
  • Pale Moon for Pandora by ptitSeb.
    A special build of Pale Moon for the OpenPandora OS on Pandora boxes.
  • Pale Moon repositories for Debian and Ubuntu by Steve Pusser.
  • Pale Moon repacks for Debian and Ubuntu by László Kovács.
  • Pale Moon COPR repositories for Fedora and CentOS by bgstack15.
  • Pale Moon for Debian MEPIS by Steve Pusser and Debian MX-15 by Mike Purtell.
    Specific build for the MEPIS/MX Linux distro.
  • Pale Moon for AOSC OS by AOSC Development Hub.
    A hardened Pale Moon build configured for AOSC OS optimized for modern personal computer architectures. Available in .deb and .rpm packages, including all supported languages with automatic selection.
  • Pale Moon for Slackware by Khronosschoty.
Historical/archived listings:

These are maintained for permissions for past contributions. You should not use these builds because the maintainers have stopped providing updates.
  • AVX build (x86/x64) by Accel.
    Note: this requires an AVX-capable CPU (later generations Intel or AMD)
  • SSE build (x86) by Roman Štefko.
    A rebuild of Pale Moon from official source catering to particularly old processors that don't support SSE2.
  • Pale Moon PPA for Debian/Ubuntu and compatible by Marián Kadaňka
    A build specifically for this OS. Source with necessary modifications available from the same page.
  • Pale Moon for Raspberry Pi boards by dimag0g
    A special build of Pale Moon for Raspberry Pi, Raspberry Pi 2, Banana Pi, Orange Pi, Odroid and related boards (ARM v6 and v7).
  • SSE build (x86) by Mercury.
    A rebuild of Pale Moon from official source catering to particularly old processors that don't support SSE2.
  • IA-32 build (x86) by Mercury.
    A rebuild of Pale Moon from official source catering to very old processors that don't support SSE.
  • Pale Moon for Windows XP by Matt A. Tobin.
    Our old, Windows XP specific builds of Pale Moon.
  • Pale Moon for Slackware by Ketmar.
    Some Slackware-specific rebuilds of old Pale Moon versions.
  • Linux SSE build by Walter Dnes.
    A rebuild of Pale Moon from official source catering to particularly old processors that don't support SSE2.
    This build has been discontinued with the advent of Pale Moon 28 and building on UXP.
Revoked listings:

These have, for one reason or another (e.g. stability, bad configs, malware) had their permissions to use official branding revoked. You should not use these builds because they have, for valid reasons, shown to not uphold the principles or quality that should be associated with official release builds of Pale Moon.

To be listed here, you have to contact me and get permission. It is the only way to be authorized to redistribute a modified binary version of Pale Moon with the official name and logo in any way, shape or form. Binaries are subject to the proprietary Pale Moon redistribution license. Please read this license thoroughly if you want to redistribute Pale Moon in any way.
You will have to supply me with a copy of the browser as-intended for distribution for me to check.

Contributed builds must:
  • Only include neutral content, i.e. not be loaded with vendor/advertiser specific extensions
  • Not contain malware of any kind!
  • Not be altered significantly after approval has been given without sending it in again for a re-check.
