Pale Moon unstable releases - release notes

These release notes are for pre-release versions of Pale Moon. They may include notes for public betas and the "rolling" unstable channel.
Please understand that these release notes are concise summaries, nothing more, and not an exhaustive record of all changes.

If you find bugs in these versions, please report them in the issue tracker on GitHub for UXP. When you do so, please check if the bug has already been reported before by others to prevent duplicate entries in the tracker. If you report bugs, make sure to include the build date (as found in the about box) and provide clear information about the bug:
  • Clear indication of what version of the browser this was found in (x86, x64, Windows, Linux, exact version and build (see "About" box))
  • Clear description of the bug found, with a concise and clear title for the issue
    (include exact circumstances and exact error messages, if any)
  • Steps to reproduce
    (what exactly do you do to cause the bug to become apparent?)
  • Expected results
    (given the steps taken, what did you expect the browser to do?)
  • Actual results
    (what did the browser do instead of what was expected?)
Also make sure that before you report a bug, that you verify if the bug also manifests with default preferences and no installed add-ons (preferably create a new profile to check).

Current trunk progress, milestone alpha 4+ (major changes only):

We have switched over to the new milestone, and these unstable builds are building on UXP. This means the browser will be significantly different under the hood than our current v27.* release versions.
  • Ported a libopus upstream patch for CVE-2017-0381.
  • Blocking of top-level data: navigations is now in effect.
  • HSTS priming has been removed for mixed content.
  • The Social API code has been removed.
  • Fixed some theme icon issues.
  • Fixed a window restore issue when on HiDPI and device pixels per px was set to a custom value.
  • Removed UITour and its SelfSupport hints and tips back-end.
  • Added a build config option to allow disabling of precompilation of the startup cache for uncommon cross-architecture build environments.
  • Enabled automatic updating properly for unstable on Linux.
  • Fixed a crash with CSP tags in chrome.
  • Fixed shellservice to work properly for integration with the OS.
  • Updated ReaderView components.
  • Fixed the quicksearch hotkey feature.
  • Fixed "Open location" when the navigation toolbar is hidden.
  • Updated NSS+NSPR and enabled TLS 1.3 by default.
  • Updated time zone data.
  • Improved several things in the JavaScript engine.
  • Applied several security/stability fixes.
  • Reinstated string.prototype.contains() as an alias for .includes().
  • Updated kiss-fft to 1.4.0.
  • Improved the 2d scaler for SSSE3.
  • Pale Moon Sync and the Sync client should now be operational again -- TESTING NEEDED.
  • Fixed regression with word-select selecting trailing space.
  • Fixed bustage if devtools was disabled at build time.
  • Changed DoNotTrack header to a simple "on/off" (since in effect the "yes please track me" header is the same as not sending anything).
  • Fixed some links on Linux.
  • Fixed lightweight themes not remaining applied after browser restart.
  • Fixed memory leak in the WebP decoder.
  • Added architecture detection in about:support.
  • Fixed the findbar (now operates on a per-tab basis).
  • Removed UA override for Google fonts since we support unicode ranges on all OSes now.
  • Fixed the size for forcing GDI classic for problematic font families.
  • Added findbar and search undo buffer sanitation to clear history.
  • Fixed some loading issues for downloads (save as...).
  • Fixed devtools strings.
  • Fixed permissions manager and cookie handling.
  • Fixed misc branding regressions in the locale.
  • Fixed SSL status ambiguity regression -- cipher name now is again the name, not the full suite string. Suite string has been made available like before in Pale Moon.

For a more detailed and complete list of changes, check out the commit log.
