Pale Moon source code

This page is intended mainly for people interested in the code changes and people needing to audit the browser code for security purposes. As a regular user, you don't need the source code of the browser.

If you want to contribute: These source code archives are snapshots of the actual releases of Pale Moon, and since it is being actively developed, any patches you may want to contribute should usually not be made against these source code archives, but against the working development branch instead. Hints for building Pale Moon yourself

If you want to build Pale Moon from source yourself, a few pointers:
  • For Windows, you will need Visual Studio 2012 (either the free Express version or the full commercial package). Even though it may possibly build with VS 2010, there will likely be some snags. It will not properly build on versions of Visual Studio older than VS2010.
    In its current state, Pale Moon will not build on VS 2013 Community or Full due to breaking changes in the MSVC compiler.
  • For Linux, you will need to use GCC (clang is not currently producing stable builds on Linux), 4.7.* recommended but later versions might work too. We do not support particularly old versions of GCC (< 4.6), and 4.9 might not like our code. GCC v5.* should work, v6.* is experimental and not recommended (YMMV).
  • You will need to get the build prerequisites as outlined on the Mozilla Developer Network. Most notably, you need the Mozilla Build Tools package (1.8-1.10, other versions may not work) and DirectX SDK if building on Windows.
  • Pale Moon should not be built with PGO (Profile-Guided Optimization). Doing so will likely not build or at the very least result in an unstable browser.
  • Do not build tests. Building tests may or may not work on your platform/configuration, and significantly increases build time for no gain unless you plan on doing automated testing with the Mozilla test harness (we don't provide support for using this test harness).
  • Pale Moon has been primarily designed for Windows and Linux, and the necessary default-theme changes were made to the relevant themes only. If you want to build for other platforms (e.g. Mac) then be aware that the browser will look different and may need some serious tweaking to be usable or look similar to the Windows/Linux version.
