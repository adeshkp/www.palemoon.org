SH bug 20110409

Browser keeps asking to be default on Vista and above.

Reporters: n/a

Affects the following:
Builds: All builds.

Cause: Hard-coded name "Firefox" as app name in the shell handler.

Details:
nsWindowsShellService.cpp has a REG_APP_NAME of L"Firefox" (hardcoded).
This mismatches the registry name for Pale Moon, as set in the branding files.

Solution:
Hardcode "Pale Moon" instead.

Implementation:
Source edit nsWindowsShellService.cpp
