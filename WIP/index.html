Pale Moon Works in Progress

Important: these builds for testing purposes only and are at all times used at your own risk. They may be unstable and may crash, do unexpected things, erase your files or set your hair on fire. Do NOT expect release stability from these!

That being said, if a file is here for more than a few days, it's likely stable enough for daily use.

These files are mainly here for beta testing before release, or to test out new and/or changed functionality in the browser, as well as tryouts of other Mozilla packages or for other Operating Systems, or work-in-progress public milestone betas.

If you wish to give feedback to these works-in-progress, please use the forum (see below for links to dedicated threads).

  • Pale Moon 26.0.0 "Goanna", public beta 1

    This is the first public beta of the new milestone release. Considering the large amount of changes involved, there will be some issues with websites, although at the moment they seem to be mostly centered around useragent sniffing (Especially Google will complain).

    This beta adds support for WebP, has faster memory handling on current Windows operating systems (at the expense of some increased memory use), and should handle particularly poorly-designed and/or image-heavy sites (e.g. Flickr) better, as main (non-inclusive) improvements to the browser.

    This beta's feedback stage has now been closed. A new beta will be released soon, implementing this feedback and other improvements.

    To provide feedback on this browser, please use the dedicated support topic on the forum. Unfortunately we cannot accept any feedback through other channels because it would result in utter chaos for us. So please, do not use e-mail, do not use social media, do not use IRC to provide feedback to this beta, and help keep things organized for us! :)

    Known issues on sites:
    • Google does not recognize Goanna and will complain and/or offer limited features and/or switch to "basic HTML" in some of their services (GMail, YouTube, Google Docs, etc.). This seems to be caused by useragent sniffing they do (tisk tisk, Google). Feel free to let Google know about the browser and ask them to support it.
    • vBulletin-based fora don't seem to like this browser, resulting in non-functional editing areas and toolbars. We're trying to figure out exactly what is causing these issues, but at first glance it seems to also be a useragent issue.
    • FaceBook seems to offer limited functionality (unsizeable comment boxes, for example) - once more determined to be a useragent sniffing issue, and this beta not having a working useragent override in place for them. Here, too, feel free to contact FaceBook support, and let them know about the browser and ask them to support it. Unfortunately their rate of responsiveness so far for Pale Moon support has been very low.
    • Some extensions that check appInfo.platformVersion may be dysfunctional, because it's currently exposed as "1.0" to reflect the 1.0 version of the Goanna platform. We do plan to work around this but it's not implemented in this first beta yet.
    • If you find other major issues, please let us know so we can add them here.
